Vue 模板语法

1.插值表达式

2.指令(属性) 增强html的功能

（1）v-cloak 指令

用法：解决插值表达式存在的闪动问题
原理：
1.代码是从上向下执行的，当加载到#app的时候，由于提供了一个V-cloak
的属性，并且属性选择器添加了display:none 的样式 ，所以是隐藏的
2.在vue初始化成功之后，此时{{msg}} 已经变成了hello world，这个时候再把v-cloak
移除掉，这时候就会显示出来
3.由于用户看到的直接就是hello，所以就不存在总{{msg}}到hello的切换

(2)v-text , v-html , v-pre

- v-text : 填充纯文本(比插值表达式更加简洁)
- v-html : 可以填充html代码(不推荐，有xss攻击的漏洞)
- v-pre  : 原样输出 会直接输出{{msg}} 不会输出hello

(3)数据响应式

页面数据会根据数据的变化，而变化

v-once 如果显示的额数据不需要修改，就使用v-once，来减少vue对数据的监听
提高性能(使数据不再响应式)

(4)双向数据绑定

v-model 绑定数据
原理: 事件监听 DOM Listeners
      数据绑定 Data Bindings

MVVM设局思想
M(model 数据)  V(view 视图) VM(View-model )
使M与V层分离 使业务变简单

(5)事件绑定

v-on (简写 @)
事件函数的调用方式
- 1 直接函数名 (如果不需要传参，用这个)
<button @click='handle'>点击2</button>
- 2 函数调用 (如果需要传参，用这个)
<button @click='handle1(3)'>点击3</button>

如果要传具体的事件对象 
直接函数名的方式是默认会带事件的对象过去
函数调用的话  必须要命名为$event

事件修饰符
.stop 阻止冒泡
.prevent 阻止默认行为

按键修饰符
.enter 回车键 （.13 也可以）

表单域修饰符
.number 转换为数值
.trim 去掉开头和结尾的空格
.lazy 将input事件变为 change事件 （v-model默认用input事件）
（input事件一改变就触发 change事件 失去焦点时才触发）

(6) 属性绑定

v-bind (简写   : )
<div v-bind:属性名 = 'data中的变量'></div>
<div :属性名 = 'data中的变量'></div>

(7) v-model 原理：

<input type='text' v-bind:value = 'msg'  v-on:input='msg = $event.target.value'>
