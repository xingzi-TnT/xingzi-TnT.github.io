Vue学习第二天

Posted on 2019-10-23

分支结构：

v-if  控制元素是否渲染到页面
v-else 
v-else-if

v-show  控制元素是否显示 （已经渲染到页面）  原理：display

<div v-if="isActive"></div>
<div v-show="isActive"></div>

循环结构：

v-for  （遵循就地更新）

因为就地更新 是只是将元素修改（重绘） 并没有将全部元素删除并创建
所以如果原始的带有比如 checkbox 就会出现问题 

解决：添加 :key 给vue一个提示。以便它可以跟踪每个节点的身份

v-for 和 v-if 一起使用 (不推荐)
当vue处理指令时,v-for 比 v-if 有更高的优先级
尽管效果是一样的 但是在内部 v-for 会循环完，性能较低

解决：先用 计算属性数据处理  从而避免 v-if 和 v-for 嵌套

计算属性：

1、计算属性解决的问题是：插值表达式太过于复杂
2、计算属性和数据模型一样，都是响应式的
3、注意事项：计算属性虽然是一个方法，但是不需要加上小括号，这是语法规范，没有为什么，应该可以理解的，
毕竟前面的事件对应的方法也是没有加括号的

计算属性和与方法的区别
    用途：
        方法：事件需要用方法
        计算属性：一个值需要各种计算 的结果  优点: 有缓存

        计算属性是基于它们的依赖来缓存的 (数据变化才变化)

监听器：

应用场景： 
当需要在数据变化时执行异步或开销较大的操作时，这个方式是最有用的。 (同步写计算属性)

var vm = new Vue({
    el: '#app',
    data: {
        msg: 'hello vue',
        list: []
    },
    methods: {},
    // 8个生命周期钩子函数
    beforeCreate: function () {
        //    console.log(this.$el)
        //    console.log(this.msg)
    },
    created: function () {
        // console.log(this.$el)
        // console.log(this.msg)
    },
    beforeMount: function () {
        // console.log(this.msg)
        // console.log(this.$el.querySelector('#ceshi').innerHTML)
    },
    mounted: function () {
        // console.log(this.msg)
        // console.log(this.$el.querySelector('#ceshi').innerHTML)
    },
    beforeUpdate: function () {
        // console.log(this.msg)
        // console.log(this.$el.querySelector('#ceshi').innerHTML)
    },
    updated: function () {
        // console.log(this.msg)
        // console.log(this.$el.querySelector('#ceshi').innerHTML)
    },
    beforeDestroy: function () {
        console.log(123)
    },
    destroyed: function () {
        console.log(456)
    }
});
```
代码执行越早越好

1. created 数据准备好 会在这个阶段可以去ajax请求数据
2. mounted DOM准备好 有些情况是必须要操作DOM 用一些第三方插件 swiper.js...
 3. beforeDestory 扫尾工作 当前组件要销毁， 用到定时器 clearTimeou
