Vue Router

Vue Router 基本使用

  • 第一步、肯定要引入vue-router.js库文件

  • 第二步、既然是路由，肯定至少要两个组件进行切换

    var Register = {

        template: `
        <div>注册组件</div>
    `
}
var User = {
    template: `
        <div>用户组件</div>
    `
}

  • 第三步、路由是一个对象，我们要new一个对象出来，并且要挂到vm实例对象 上面，与vue实例对象产生关系

var vm = new Vue({
el: ‘#app’,
data: {},
methods: {},
router:new VueRouter
});

  • 第四步、路由就是设置hash与组件之间的映射关系

new VueRouter({
routes: [{
path: ‘/register’,
component: Register
},
{
path: ‘/user’,
component: User
},
]
})

  • 第五步、需要放一个占位符 (将来会被组件所替换)

    <router-view></router-view>

  • 第六步、手动输入太麻烦了，可以设置vue-router专用链接router-link

    <router-link to="/register">register</router-link>
<router-link to="/user">user</router-link>

路由重定向

开发中有时候我们虽然设置的路径不一致，但是我们希望跳转到同一个页面，或者说是打开同一个组件。这时候我们就用到了路由的重新定向redirect参数。

  routes: [{
                path: '/register',
                component: Register
            },
            {
                path: '/user',
                component: User
            },
            {
                path: '/',
                redirect: '/user'
            }
        ]     
   这里我们 打开默认首页 会重定向到/user

嵌套路由

所谓的嵌套路由就是子路由，也就是路由中的路由

(1)在路由规则中添加子路由

     routes: [{ //路由匹配规则 
                // path 表示监听 那个路由连接地址
                // component 表示路由皮牌到那个地址，则展示component属性对应的那个组件 
                path: '/register',
                // 动态路径参数  以冒号开头
                component: Register,
                children: [{
                    path: '/register/tab1',
                    component: Tab1,
                }, {
                    path: '/register/tab2',
                    component: Tab2,
                }]
            }

(2)在父级模板中加入占位符和子模板
    var Register = {
        template: `
        <div>
            <div>注册组件</div>
            <hr>
            <router-link to="/register/tab1">tab1</router-link>
            <router-link to="/register/tab2">tab2</router-link>
            <transition mode='out-in'>
            <router-view></router-view>
            </transition>
        </div>    
        `,
    }

动态匹配路由

{
    path: '/user/:id',
    component: User,
    props: true
},

布尔模式
 var User = {
    props: ['id'],
    template: `
    <div>
        <transition mode='out-in'>
        <div>用户组件 --- 用户id为：{{id}}</div>
        </transition>
        <hr>
        <router-link to="/user/1">user1</router-link>
        <router-link to="/user/2">user2</router-link>
        <router-link to="/user/3">user3</router-link>
    </div>`
}

函数模式
{
    path: '/user/:id',
    component: User,
    props: function (route) {
        return {
            name: "zs",
            age: 14,
            id: route.params.id
        }
    }
},

用箭头函数的话,在对象外面加小括号，提高优先级
不然会被当成函数体来解析

{
    path: '/user/:id',
    component: User,
    props:route => ({name: "zs",age: 14,id:route.params.id})
},

命名路由

为了更加方便的表示路由的路径，可以给路由规则起一个别名
 {
    path: '/user/:id',
    component: User,
    name: 'user',
    props: function (route) {
        return {
            name: "zs",
            age: 14,
            id: route.params.id
        }
    }
},   

注意： 格式  :to="{name:'user',params:{id:1}}":to="{name:'user',params:{id:1}}"
 var User = {
        props: ['id'],
        template: `
        <div>
            <transition mode='out-in'>
            <div>用户组件 --- 用户id为：{{id}}</div>
            </transition>
            <hr>
            <router-link :to="{name:'user',params:{id:1}}">user1</router-link>
            <router-link :to="{name:'user',params:{id:2}}">user2</router-link>
            <router-link :to="{name:'user',params:{id:3}}">user3</router-link>
        </div>`
    }
