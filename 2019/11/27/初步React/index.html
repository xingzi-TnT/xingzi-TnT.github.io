React 事件处理

React 元素的事件处理和 DOM 元素类似。但是有一点语法上的不同:

React 事件绑定属性的命名采用驼峰式写法，而不是小写。
如果采用 JSX 的语法你需要传入一个函数作为事件处理函数，
而不是一个字符串(DOM 元素的写法)

在React 中不能用 return false 来阻止默认行为
必须明确使用 e.preventDefault

JSX 回调函数中的 this，类的方法默认是不会绑定 this 的。 ！！

API

setState(object nextState[, function callback])
参数说明
nextState，将要设置的新状态，该状态会和当前的state合并
callback，可选参数，回调函数。该函数会在setState设置成功，
且组件重新渲染后调用。
合并nextState和当前state，并重新渲染组件。setState是React事件处理
函数中和请求回调函数中触发UI更新的主要方法。

React 父子组件传值

子级向父级 传值：

子组件向父组件传递数据：子组件绑定一个方法，
方法中通过this.props.父组件方法名（参数）传递给父组件，
父组件通过该方法接受数据。

子组件中传递数据：
<button onClcik={()=>{this.change(value)}}>

change=(value)=>{
this.props.handleClick();
}

父组件中接收数据：

handleClick=(value)=>{
console.log(value)
}

Redux

redux 的三个部分 action reducer state(store)

action 通过action来进行与外部的通信 (就是进入仓库的门票)

reducer 仓库的员工 (具体进行什么修改)

Redux 原则 ： 单项数据流，不允许直接修改state(只读)，使用reducer 纯函数来进行修改

流程：
先从 Redux 接收 createStore

再定义一个 reducer 函数 (传入初始 的状态 和 action)

function(state=0,action){
switch(action.type){
case:”add”:
return state + 1
case:”sub”:
return state - 1
default :
return state
}
}

import { connect } from ‘react-redux’

然后通过 connect 与 state(仓库连接)
export default connect()(App);

再通过 dispatch({type:’’}) 来派发 action 的执行
可以根据 reducer 改变 state 值

如果要取出 state 的值 要增强 connect

connect 可以传两个参数

connect(mapStateToProps,)

mapStateToProps 映射

const mapStateToProps = state => {
return {
要映射的值:state
}
}
